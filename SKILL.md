# Price Comparison Skill (RU) v5.0 — Category Templates + Inline Eval + Dual Analogs

## Назначение
Поиск и сравнение цен на материалы и оборудование у российских поставщиков. Работает с Excel-таблицами, ищет по B2B-каталогам и маркетплейсам, записывает 2 цены от разных поставщиков. **Включает автоматическую проверку (eval) результата перед выдачей.**

**Новое в v5.0:** Inline eval (проверка в процессе, не пост-фактум) + шаблоны категорий (5-8 параметров вместо 30+) + два типа аналогов с красной командой для каждого.

**Новое в v4.1:** Цель — не просто сравнить цены, а **снизить цену** через аналоги (включая китайские и внутрибрендовые с незначительными отклонениями).

**Новое в v4.0:** Три типа аналогов (оригинал, другая марка, та же марка с отклонениями) + стандартизированный Excel-формат с вкладкой «Отклонения [Марка]».

**Новое в v3.0:** Для каждого найденного аналога создаётся отдельная вкладка Excel с технической сравнительной матрицей.

## Цель и приоритеты

1. **Снизить цену** — главная цель, не просто «сравнить»
2. **Найти аналоги** — другая марка (включая китайские) или та же марка с отклонениями
3. **Согласовать с заказчиком** — подготовить обоснование, почему отклонение некритично
4. Сохранить цветовую схему и форматирование из оригинального Excel

## Шаблоны параметров по категориям (v5.0)

**Новое:** Агент автоматически определяет категорию ТМЦ и загружает шаблон с нужными параметрами. Не собирает 30+ строк из datasheet — только то, что реально важно для этой категории.

### Как работает

1. **Автоопределение категории** — агент берёт название ТМЦ, ищет ключевые слова в `templates/category_map.json`
2. **Загрузка шаблона** — из `templates/categories/[category].json` берётся список `key_params`
3. **Сбор данных** — агент собирает **только** параметры из шаблона (не всё подряд)
4. **Обучение** — при первой встрече новой категории собирается полная матрица, пользователь помечает критичные, шаблон обновляется

### Готовые категории

| Категория | Файл | Статус | Параметров |
|-----------|------|--------|------------|
| IP-камеры | `ip_cameras.json` | ✅ Готов | 7-10 (из 30+) |
| Оптика (кассеты, патч-корды) | `fiber_optic_patches.json` | ✅ Готов | 8 |
| Медные кабели | `copper_cables.json` | ⚪ Базовый | 6 |
| Коммутаторы | `switches.json` | ⚪ Базовый | 7 |
| Неизвестно | `_unknown.json` | ✅ Заглушка | Собирает всё |

### Формат шаблона (JSON)

```json
{
  "category": "ip_cameras",
  "key_params": {
    "resolution": {
      "label": "Разрешение",
      "example": "5Мп (2592×1944)",
      "critical": true,
      "note": "Критично для детализации"
    }
  },
  "version": 1,
  "learned_from": ["Beward SV3218RBZ"]
}
```

### Правило применения

- **Если категория найдена** — собирать только `key_params` из шаблона
- **Если категория не найдена** — использовать `_unknown.json` (собирать всё), после сохранить новый шаблон
- **После согласования с заказчиком** — пользователь обновляет `critical: true/false` в шаблоне, инкрементирует `version`

### Добавление новой категории

1. Создать `categories/[new_category].json` по образцу
2. Добавить ключевые слова в `category_map.json`
3. При первом использовании — агент собирает всё, шаблон обновляется из практики

## Три типа аналогов (v3.1)

| Тип | Описание | Когда искать |
|-----|----------|-------------|
| **Оригинал** | Тот же артикул/модель у разных поставщиков | Всегда — минимум 2 цены |
| **Аналог другой марки** | Другой бренд, идентичные технические параметры | Если оригинал дорогой или под заказ |
| **Аналог той же марки с отклонениями** | Тот же бренд, другая модель/артикул, есть отклонения от оригинала | Если пользователь явно запросил или не нашлось других вариантов |

### Критерии отклонений

**Незначительные отклонения** (не влияют на функциональность):
- Цвет корпуса
- Вес/габариты (±20%)
- Страна производства
- Гарантийный срок
- Упаковка/комплектация

**Значительные отклонения** (влияют на совместимость или производительность):
- Полярность (Type A vs B)
- Тип волокна (OM3 vs OM4 vs OS2)
- Количество портов/волокон
- Тип коннектора (APC vs UPC vs PC)
- Форм-фактор (не совместим с шасси/панелью)
- Питание (PoE vs без PoE)
- Скорость интерфейса (1G vs 10G)
- Тип SFP+/QSFP (разные стандарты)

### Правило поиска аналога той же марки с отклонениями

1. Искать в каталоге того же бренда похожие модели
2. Сравнить спецификации оригинала и аналога
3. Если найден за **10 минут** — добавить в отчёт
4. Если **не найден за 10 минут** — прекратить поиск, в комментарий записать: «Аналог той же марки с отклонениями не найден за 10 минут»
5. **Обязательно** создать вкладку «Отклонения [Марка] [Модель]» с таблицей:

| Параметр | Оригинал | Аналог (та же марка) | Тип отклонения | Влияние |
|----------|----------|---------------------|----------------|---------|
| Пример: Полярность | Type A | Type B | Значительное | Несовместим с trunk-кабелем |

6. В комментарий к позиции добавить сводку: «Аналог той же марки: [модель]. Отклонения: [список]. [Рекомендация]»

## Формат выходного Excel (v4)

Обязательная структура файла (пример на основе Nikomax NMF-CT12M4PA):

### Вкладка 1: Результат сравнения

| № | Наименование | Цена 1 (₽) | Поставщик 1 | URL 1 | Цена 2 (₽) | Поставщик 2 | URL 2 | Аналог (другая марка) | Цена аналога (₽) | Поставщик аналога | URL аналога | Комментарий |
|---|-------------|-----------|------------|-------|-----------|------------|-------|----------------------|-----------------|------------------|------------|-------------|
| 1 | Nikomax NMF-CT12M4PA... | 25862 | Regard.ru | ... | 22353 | Born-shop.ru | ... | SNR MCM4-MPO... | 7996 | Born-shop.ru | ... | Аналог другой марки: SNR... |

**Требования:**
- Минимум 2 цены оригинала от разных поставщиков
- Аналог другой марки с совпадающими техническими параметрами
- Все URL кликабельные (http/https)
- Цены в числовом формате

### Вкладка 2: Отклонения [Марка]

Создаётся, если найден аналог **той же марки** с отклонениями.

| Параметр | Оригинал | Аналог 1 | Аналог 2 | ... | Тип отклонения | Влияние |
|----------|----------|----------|----------|-----|---------------|---------|
| Цена | 22 353 ₽ | 19 674 ₽ | 25 133 ₽ | ... | — | Экономия / переплата |
| Класс волокна | OM4 | OM3 | OM4 | ... | Значительное | Меньшее расстояние 10G/40G |
| Полярность | Type A | Type A | Type B2 | ... | Значительное | Несовместим с trunk-кабелем |

**Требования:**
- Одна вкладка на бренд (если несколько позиций одного бренда — в одной вкладке)
- Явное указание: «Значительное» или «Незначительное» отклонение
- Описание влияния на функциональность

### Вкладка 3: Сводка

| Критерий | Значение |
|----------|----------|
| Цена оригинала | Диапазон цен |
| Цена аналога (другая марка) | Цена + экономия % |
| Аналоги той же марки | Список с отклонениями |
| Рекомендация | Какой вариант выбрать |
| Риски | Что проверить перед заказом |

**Цветовая схема:**
- Заголовки: синий фон (#4472C4), белый текст
- 🟢 Совпадает / лучше
- 🟡 Частичное совпадение
- 🔴 Критичное отличие
- ⚪ Не найдено / не применимо

## Порядок работы (v3.0)

### Шаг 0: Подготовка и поиск цен

1. Прочитать входной Excel-файл
2. Для каждой позиции найти:
   - Цена 1 (та же марка, поставщик 1)
   - Цена 2 (та же марка, поставщик 2)
   - Цена аналога (другая марка, ближайшие характеристики)
3. Записать данные в JSON

```json
[
  {
    "num": 1,
    "name": "Cisco C9200-24P-E",
    "price1": 106714, "supplier1": "shop.nag.ru", "url1": "...",
    "price2": 182112, "supplier2": "ediscom.ru", "url2": "...",
    "analog": "Huawei S5735-S24P4X",
    "analog_price": 99194, "analog_supplier": "network.msk.ru", "analog_url": "...",
    "comment": "..."
  }
]
```

**Авто-определение бренда:** Бренд определяется автоматически из первого слова названия позиции. Например, "Cisco C9200-24P-E" → бренд «Cisco», "Beward SV3218RBZ" → бренд «Beward». Заголовки колонок формируются динамически: «Цена 1 (₽) Cisco, без НДС».

**Если аналог не найден за 5 минут:**
- В колонке «Аналог (другая марка)» писать: «Аналог не найден»
- Колонки цены аналога, поставщика, URL аналога оставить пустыми
- В «Комментарий» указать: «Аналог не найден за 5 минут поиска. [причина: проприетарный форм-фактор / уникальная технология / отсутствие на рынке РФ]»

### Шаг 1: Создание основной таблицы

```bash
python3 skills/price-comparison/scripts/runner_v2.py input.xlsx results.json
```

Скрипт создаёт:
- `price_comparison_main_*.xlsx` — основная таблица с ценами
- `analogs_to_research_*.json` — список аналогов для субагентов

### Шаг 2: Запуск субагентов для аналогов

Для КАЖДОГО аналога из `analogs_to_research_*.json` запустить отдельного субагента.

**Параметры для субагента:**
- `original` — название оригинала
- `analog` — название аналога
- `original_price` / `analog_price` — цены (опционально)

**Задача субагента** (полный текст в `subagent-task.md`):
1. Поискать технические характеристики обоих устройств через `kimi_search`
2. Сравнить по 8 категориям (порты, питание, производительность, ПО, физика, безопасность, интеграция, надёжность)
3. Для каждого параметра установить статус: ✅/⚠️/🔴/⚪
4. Сохранить результат в `analogs/analog_matrix_[original]_[analog].json`

**Параллельный запуск:** Если аналогов несколько, субагенты запускаются параллельно.

### Шаг 3: Сборка финального Excel с матрицами

```bash
python3 skills/price-comparison/scripts/matrix_builder.py \
    price_comparison_main_*.xlsx \
    analogs/ \
    output_final.xlsx
```

**Структура выходного Excel:**

| Вкладка | Содержимое |
|---------|------------|
| `Сводка аналогов` | Общая сводка (экономия, рекомендации) |
| `Результат сравнения` | Основная таблица с ценами |
| `Матрица [Аналог 1]` | Детальная техническая матрица (30+ параметров) |
| `Матрица [Аналог 2]` | Детальная техническая матрица |
| ... | ... |

**Цветовая кодировка матрицы:**
- 🟢 Зелёный — характеристики совпадают или аналог лучше
- 🟡 Жёлтый — частичное совпадение, есть отличия (не критичные)
- 🔴 Красный — критичное отличие, функциональность потеряна
- ⚪ Серый — информация не найдена

### Шаг 4: Inline Eval v5 — проверка в процессе (НЕ пост-фактум)

**Старый подход (v4):** eval.py проверяет готовый Excel в конце. Если FAIL — переделывать всё.

**Новый подход (v5):** проверки встроены в каждый шаг. Ошибки ловятся сразу, не накапливаются.

#### Поток inline eval

```
Поиск цены 1 ──► Inline check ──► FAIL? → Исправить сразу
      │
      ▼
Поиск цены 2 ──► Inline check ──► FAIL? → Исправить сразу
      │
      ▼
Аналог другой марки? ──► Inline check ──► FAIL? → Искать другой
      │
      ▼
Аналог той же марки? ──► Inline check ──► FAIL? → Пометить отклонения
      │
      ▼
Матрица по шаблону категории (5-8 параметров)
      │
      ▼
Красная команда (проверка критичных параметров)
      │
      ▼
Комментарий: [Цена] [Дата] [Риски] [TCO] [Рекомендация]
      │
      ▼
Запись в Excel → Следующая позиция
```

#### Inline check: Оригинал (цены)

| Проверка | Если FAIL | Действие |
|----------|-----------|----------|
| Найдена ≥1 цена? | Нет | Продолжить поиск, не переходить к аналогам |
| Цена — число? | Текст ("договорная") | Перевести в число или пометить ⚠️ |
| URL кликабельный? | Нет | Перепроверить, найти прямую страницу |
| Цены отличаются >10x? | Да | Остановиться, спросить: «правильный ли артикул?» |
| Поставщики разные? | Один и тот же | Искать второго поставщика |

#### Inline check: Аналог другой марки

| Проверка | Если FAIL | Действие |
|----------|-----------|----------|
| Марка аналога ≠ марка оригинала? | Та же марка → это не аналог другой марки | Переклассифицировать как «аналог той же марки» или искать дальше |
| Найден за 5 мин? | Нет | Записать «Не найден», перейти к аналогу той же марки |
| Цена аналога < оригинала? | Дороже | В комментарий: «аналог дороже, требуется обоснование» |

#### Inline check: Аналог той же марки с отклонениями

| Проверка | Если FAIL | Действие |
|----------|-----------|----------|
| Найден за 10 мин? | Нет | Записать «Не найден», остановить поиск |
| Отклонения классифицированы? | Нет | Проверить: значительное / незначительное |
| Есть значительные отклонения? | Да | Graceful degradation: fallback к оригиналу или другому аналогу |
| Создана вкладка «Отклонения [Марка]»? | Нет | Создать перед записью в Excel |

#### Красная команда (для ОБОИХ типов аналогов)

После заполнения матрицы — проверка «почему НЕ подходит»:

```
Для аналога другой марки:
  → Проверить 3-5 критичных параметров из шаблона категории
  → Найти: что сломается, если поставить этот аналог?
  → Записать в комментарий: ⚠️ или ❌

Для аналога той же марки:
  → Проверить: отклонения значительные?
  → Если да: объяснить последствия
  → Записать в комментарий: рекомендация (согласовать / не согласовывать)
```

**Пример результата красной команды:**
```
Аналог другой марки: SNR MCM4-MPO-1U-12LC-AQU
⚠️ Внимание: класс волокна OM3 (vs OM4 оригинал) — 
   расстояние 10G сокращается с 400м до 300м.
   Для текущего проекта (кросс 50м) — не критично.
   Рекомендация: согласовать с заказчиком.

Аналог той же марки: Beward SV3210RBZ
✅ Все критичные параметры совпадают.
⚠️ Отклонение: объектив 2.8-11мм (vs 2.7-13.5мм) — 
   незначительное, зона покрытия сокращается на 15%.
   Рекомендация: согласовать.
```

#### Структура комментария (для ОБОИХ типов аналогов)

```
[ЦЕНА] Оригинал: 22 353 ₽. Аналог другой марки: 7 996 ₽ (экономия 64%).
       Аналог той же марки: 19 674 ₽ (экономия 12%).

[ДАТА] 2026-07-04. Источник: born-shop.ru.

[АНАЛОГ ДРУГОЙ МАРКИ — КРАСНАЯ КОМАНДА]
⚠️ OM3 вместо OM4 — для кросса 50м допустимо.

[АНАЛОГ ТОЙ ЖЕ МАРКИ — ОТКЛОНЕНИЯ]
✅ Критичные параметры совпадают.
⚠️ Объектив уже — зона покрытия -15%.

[TCO] Экономия на закупке: 14 357 ₽ (другая марка).
      Риск: если расстояние вырастет >300м — замена.
      Вероятность: низкая (проект фиксирован).

[РЕКОМЕНДАЦИЯ] Согласовать аналог другой марки (SNR).
               Аналог той же марки — резерв, если SNR не пройдёт.
```

#### Пост-проверка (упрощённый eval)

После заполнения всех позиций — быстрая проверка:

| Проверка | Уровень |
|----------|---------|
| Количество строк = input | 🔴 FAIL |
| Все цены — числовой формат | 🔴 FAIL |
| Все URL кликабельны | 🔴 FAIL |
| Комментарий не пустой у каждой позиции | 🟡 WARN |
| Дата поиска указана | 🟡 WARN |

**Результат:**
- **PASS** — готово к выдаче
- **WARN** — можно показать с пометками
- **FAIL** — исправить конкретные позиции (не всё с нуля)

**Поток данных:**
1. **Определение категории** → загрузка шаблона (5-8 параметров вместо 30+)
2. **Поиск цен** → inline eval (число, URL, поставщики)
3. **Поиск аналога другой марки** → inline eval (марка ≠, цена, 5 мин)
4. **Поиск аналога той же марки** → inline eval (отклонения, 10 мин, значимость)
5. **Матрица** по шаблону категории
6. **Красная команда** для обоих типов аналогов
7. **Комментарий** с ценой, датой, рисками, TCO, рекомендацией
8. **Запись в Excel** → следующая позиция
9. **Пост-проверка** (быстрая, не полная переделка)

**Два типа аналогов в выходном файле:**
- **Вкладка 1:** Основная таблица с ценами + аналог другой марки
- **Вкладка 2:** «Отклонения [Марка]» — аналоги той же марки с отклонениями

## Правила защиты от ошибок

1. **Никогда не предполагать количество строк** — читать исходный файл
2. **Никогда не добавлять позиции вне списка** — никаких "полезных" аналогов
3. **Точность артикулов** — `C47-60` и `C47-40` — разные позиции
4. **URL — прямые страницы** — если URL ведёт на каталог, искать конкретный товар
5. **Снятые с производства** — маркировать, не придумывать цену
6. **Two-pass поиск** — если по артикулу не нашёл, пробовать описательный запрос
7. **Цена с НДС** — если на сайте без НДС, указывать примечание
8. **EVAL перед выдачей** — всегда проверяй результат скриптом
9. **Матрицы — обязательно** — для каждого аналога должна быть отдельная вкладка
10. **Субагенты не дублируют работу** — цены ищет основной агент, specs — субагенты
