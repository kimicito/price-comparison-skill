# Plan.md — Шаблон: Price Comparison (15+ позиций)

## Задача
Поиск и сравнение цен на [N] позиций оборудования/материалов.

## Этапы

### 1. Анализ (30 мин)
- [ ] Получить Excel/список позиций
- [ ] Определить категорию каждой позиции (ip_cameras, switches, cables, etc.)
- [ ] Проверить кэш (актуален? очистить если нет)
- [ ] Оценить сложность: сколько аналогов нужно?

### 2. Подготовка (15 мин)
- [ ] Активировать venv (если есть)
- [ ] Проверить доступность API поиска
- [ ] Подготовить шаблоны категорий
- [ ] Создать results.json с заготовками

### 3. Поиск цен (основной этап)
Для каждой позиции:
- [ ] Проверить кэш
- [ ] Найти цену 1 (основной поставщик)
- [ ] Найти цену 2 (альтернативный поставщик)
- [ ] Найти аналог (другая марка)
- [ ] Найти аналог (та же марка, отклонения)
- [ ] Inline eval: цены адекватны? URL валидны?

### 4. Субагенты (для критичных аналогов)
- [ ] Определить, какие аналоги нуждаются в глубоком исследовании
- [ ] Запустить субагентов с retry + fallback
- [ ] Собрать технические матрицы

### 5. Сборка и проверка (30 мин)
- [ ] Запустить runner_v2.py → Excel
- [ ] Post-factum eval: все ли проверки пройдены?
- [ ] Sanity checks: цены не отличаются >10x? Поставщики разные?

### 6. Документирование (15 мин)
- [ ] Обновить SKILL.md если были изменения
- [ ] Записать результат в memory
- [ ] Закоммитить изменения

## Риски

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| API поиска недоступен | Средняя | Использовать кэш, fallback на ручной поиск |
| Нет аналогов | Высокая | Искать китайские аналоги, расширять категории |
| Цены устарели | Средняя | Проверять дату кэша, искать свежие |
| Субагент падает | Низкая | Retry 3 раза + fallback задачи |

## Ресурсы

- Шаблоны: `templates/categories/`
- Скрипты: `scripts/`
