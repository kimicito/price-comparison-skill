# Шаблоны параметров по категориям ТМЦ

## Как работает система

1. **Автоопределение категории** — агент берёт название ТМЦ, ищет ключевые слова в `category_map.json`, определяет категорию
2. **Загрузка шаблона** — из `categories/[category].json` берётся список параметров, которые реально важны для этой категории
3. **Сбор данных** — агент собирает только параметры из `key_params` (не все 30+ из datasheet)
4. **Обучение** — после первого использования новой категории пользователь помечает, что было критично, шаблон обновляется

## Структура шаблона

```json
{
  "category": "ip_cameras",
  "description": "...",
  "key_params": {
    "param_id": {
      "label": "Человекочитаемое название",
      "example": "Пример значения",
      "critical": true/false,
      "note": "Почему это важно / когда не важно"
    }
  },
  "auto_detect_keywords": ["..."],
  "version": 1,
  "learned_from": ["Модель 1", "Модель 2"],
  "date_created": "2026-07-04"
}
```

## Категории

| Файл | Описание | Статус |
|------|----------|--------|
| `ip_cameras.json` | IP-камеры | ✅ Готов (опыт: Beward, Dahua) |
| `fiber_optic_patches.json` | Оптика (кассеты, патч-корды) | ✅ Готов (опыт: Nikomax) |
| `copper_cables.json` | Медные кабели | ⚪ Базовый |
| `switches.json` | Коммутаторы | ⚪ Базовый |
| `_unknown.json` | Заглушка для новых | ✅ Готов |

## Добавление новой категории

1. Создать `categories/[new_category].json` по образцу
2. Добавить ключевые слова в `category_map.json`
3. При первом использовании агент собирает все параметры — пользователь помечает критичные
4. Обновить `key_params` в шаблоне, убрать лишнее
5. Инкрементировать `version`

## Обновление существующей категории

Если при согласовании с заказчиком выяснилось, что параметр оказался критичным:
1. Открыть `categories/[category].json`
2. Добавить/поменять `critical: true`
3. Обновить `note` с пояснением
4. Инкрементировать `version`
5. Сохранить копию в `learned/` с датой
